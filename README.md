## 📌 My-Portfolio
**My First Personal Portfolio Website**

---

## 📝 Short Description
A multi-page personal portfolio website built from scratch using HTML5 and CSS3 as a final showcase project for a beginner web development course.

---

## 🎯 Purpose
The goal of this website is to:
- Introduce myself to the world as a beginner web developer
- Showcase the HTML and CSS concepts I have learned throughout my course
- Display projects and exercises I have completed
- Share my learning journey, goals, and interests
- Practice building a real, structured, and styled multi-page website

---

## ✨ Features
- **Multi-page layout** — 4 pages: Home, About, Projects, Contact
- **Responsive design** — Works on desktop, tablet, and mobile
- **Fixed navigation bar** with smooth hover effects
- **Hero section** with an animated blob shape and stat counters
- **Skills section** with animated progress bars
- **Learning journey timeline** on the About page
- **Project card grid** with hover lift animations
- **HTML tables** for goals tracking and concepts summary
- **Contact form** with all input types (text, email, select, radio, checkbox)
- **Back-to-top button** that appears on scroll
- **Mobile hamburger menu** for small screens
- **Custom colour palette** and Google Fonts (Playfair Display + DM Sans)
- **CSS custom properties** (variables) for consistent theming
- **Bonus:** CSS keyframe animations, morphing blob, pulse dot, download CV button

---

## 🌐 HTML Concepts Used
- `<!DOCTYPE html>`, `<html>`, `<head>`, `<body>` document structure
- `<title>`, `<meta charset>`, `<meta name="viewport">`, `<meta name="description">` — meta tags
- Headings: `<h1>` through `<h4>` in correct hierarchy
- Paragraphs `<p>`, links `<a>`, images `<img alt="">` with alt text
- Ordered and unordered lists `<ul>`, `<ol>`, `<li>`
- Tables: `<table>`, `<thead>`, `<tbody>`, `<tr>`, `<th scope>`, `<td>`, `<caption>`
- Forms: `<form>`, `<label>`, `<input>` (text, email, radio, checkbox), `<textarea>`, `<select>`, `<option>`, `<button>`
- Semantic elements: `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`, `<blockquote>`, `<figure>`
- HTML comments (`<!-- -->`) throughout the code
- Correct indentation and readable, well-organised code
- `aria-label`, `aria-required`, `role` attributes for accessibility

---

## 🎨 CSS Concepts Used
- External stylesheet linked via `<link rel="stylesheet">`
- CSS custom properties (variables) — `--clr-primary`, `--ff-body`, etc.
- Element, class (`.btn`), and ID (`#hero`) selectors
- Colours: hex codes, rgba, CSS gradients
- Google Fonts via `@import`
- Margins, padding, borders — the full Box Model
- Backgrounds: solid colours, linear gradients, radial gradients
- Text styling: `font-family`, `font-size`, `font-weight`, `line-height`, `letter-spacing`, `text-transform`
- Image styling: `max-width`, `border-radius`, `object-fit`
- Hover effects: `transition`, `transform: translateY()`, `box-shadow`
- Flexbox: `display: flex`, `align-items`, `justify-content`, `gap`, `flex-wrap`, `flex-direction`
- CSS Grid: `display: grid`, `grid-template-columns`, `gap`, `auto-fit`, `minmax()`
- Responsive design with `@media` queries (max-width: 900px, 700px, 480px)
- `position: fixed`, `sticky`, `absolute`, `relative` — all positioning types
- CSS `animation` and `@keyframes` (morphing blob, skill bars, pulse dot, fade-up)
- `backdrop-filter` for frosted glass navbar effect
- `clamp()` for fluid typography
- Consistent spacing system
- `scroll-behavior: smooth` on `html`
- Print styles (`@media print`)

 Challenges Faced
1. **Getting Flexbox to behave** — especially centering items both horizontally and vertically took trial and error.
2. **Making the navbar sticky on scroll** — understanding `position: fixed` vs `sticky` and how they interact with page flow.
3. **Responsive design** — writing media queries and figuring out when layouts need to change was harder than expected.
4. **CSS specificity conflicts** — sometimes styles didn't apply because of selector specificity, which taught me to plan my CSS structure better.
5. **Form styling** — making default browser inputs look consistent across Chrome, Firefox, and Safari.



 📖 What I Learned
- How HTML and CSS work together to create complete web pages
- The importance of semantic HTML for readability and accessibility
- How the CSS Box Model controls all spacing and sizing
- How to use Flexbox and Grid for modern, flexible layouts
- How to use CSS custom properties to maintain a consistent design system
- How responsive design and media queries make sites work on all devices
- How to structure a multi-page project with a shared stylesheet
- How to use Git and GitHub for version control
- The value of clean, commented, well-indented code
